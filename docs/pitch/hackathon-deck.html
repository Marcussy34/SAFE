The Problem Wallet caps are not judgment.

A valid transaction can still be the wrong payment.

Spend about 15 seconds. Make the distinction clear: crypto payment validity is not the same as user-intent validity. Public x402 sources expose transactions and volume, but not a verified count of wrong agent payments. The 35k-350k estimate is a conservative model: 0.1%-1% of 35M transactions. Published x402 attack research observed 5.18% revert-grant in vulnerable configs and live replay up to 248 grants/payment, so the tail risk is credible.
The Solution Pre-signing policy layer

SAFE sits between the agent and money.

01
x402 challenge
Paid API returns a structured payment requirement.
02
Normalize
Merchant, amount, token, recipient, URL, reason.
03
Policy check
Trust, category, cap, replay, PII, allowance.
04
Settle or block
Approved payments settle. Unsafe ones never sign.
05
Audit
Decision, reason, and tx receipt.
SAFE is not a wallet. SAFE is the buyer-side payment firewall. 03 / 07
Say: "The agent does not sign directly. It asks SAFE. SAFE approves, rejects, or redacts before any settlement is attempted."
